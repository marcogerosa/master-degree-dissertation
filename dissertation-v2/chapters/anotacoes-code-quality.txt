Boas Práticas de Software
	
	Boas práticas de software ....

	É muito importante. É originada da experiência de desenvolvedores mais experiêntes.

	Podemos dizer que padrões de projetos, anti-padrões e maus cheiros de código são boas práticas de software visto que de alguma forma, ajudam a aumentar a qualidade do software.

	No dia a dia, estes termos podem facilmente se confundir, e apesar de serem todos boas práticas de software, devemos interpretá-los de forma correta por exemplo, enquanto que padrões de projeto são feitos para serem seguidos, maus cheiros de código e anti-padrões foram feitos para serem evitados.
